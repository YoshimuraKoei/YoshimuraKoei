## 得意なこと・不得意なこと

知的好奇心があり, とにかく何事も本気で取り組みます. どちらかというと, とりあえず手を動かしてどんどん改善していくタイプです. 達成したい目標に対して, 本気で取り組めるのが強みかと思います. また, 特に少人数での話し合いや, 相談に乗ることが得意であると自負しています. これは恐らく, 少人数であるからこそおそらくラリーが踏めるからこそ, 回数をこなして深掘っていけるからだと思います.

## やってきたことなど

### 📱 勤怠管理のモバイルアプリ開発
TypeScript × ReactNative の技術構成で Android, iOS 対応のネイティブアプリケーション開発をチームで行いました.
私は主にフロントエンドを担当し, Storybook を用いた UI 開発, CSS, フロントのバリデーションや勤怠ロジックなどを経験しました.
アジャイル開発の形式を取り, 2週間ごとにチームでスプリントレビューを行うのが楽しかったです. 
開発の楽しさを知ったのがこのプロジェクトで, 大学3年時でしたが, 免許合宿中でも教習の合間にコミットしていた思い出があります.  

### 📈 データ解析コンペティション
書店の販売データを対象としたデータ解析コンペティションにチームで参加しました.
最終的には, 書店・店舗の販売傾向や特徴をもとに, その店舗の特徴に合いそうな書籍を提案する推薦システムを提案しました. 
購買データだけでなく, 書籍内容や店舗特性に基づいて推薦結果を, ペルソナやジャンルの観点から解釈できる形に昇華させる点を工夫しています. 
結果は奮わなかったですが, データから何かを生み出して提案することの楽しさから, プレゼンで伝えることの難しさまで, 様々なことを学べて良かったです. 

### ☀️ 朝活 bot
完全に趣味ですが, 研究室の Slack に朝活 bot を導入しました.
「午前 8:30~9:00 の間に研究室で Slack のボタンを押すと朝活ポイントが貯まる」という仕組みにしており, 現在10名程度が z-asakatsu チャンネルに所属しています.
Slack API と GAS の完全無料技術スタック（有料プランに元から入っているのは除く）で構成されており, GAS のトリガーによって毎日 bot のメッセージが送られてくる仕様です.
ゲーム性を取り入れたことで, 「1人じゃやる気にならないけど, 皆となら頑張れる」という心理が働くようです. 小さい規模ですが, 世のため人のためとなれているプロダクトかなと思います.
